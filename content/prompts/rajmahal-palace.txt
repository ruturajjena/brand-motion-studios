# The Rajmahal Palace — Google Flow (Veo) Prompt Pack
### Scroll-driven cinematic palace-hotel website · 6 clips · max 10s each
### Style reference: The Leela Palace Udaipur / Taj Lake Palace — royal Indian heritage luxury

---

## How to use this pack

1. **Paste the Global Style Block at the END of every prompt.** Veo has no memory between generations — this block keeps the architecture, color grade and lighting consistent so all 6 clips feel like one palace, not six.
2. **Generate at the highest resolution Flow allows** (4K if available).
3. **Every clip uses ONE slow camera move in ONE direction, constant speed, no cuts.** The website will scrub these videos frame-by-frame with scroll — a cut or reversing motion will feel broken. Regenerate any clip where Flow adds a cut.
4. **No people, no text, no logos** in any clip.
5. Name exports exactly as labeled — Claude Code will wire them by filename.

The entire site is one continuous scroll experience where video, imagery and typography reveal themselves as the user scrolls — think Apple product pages meets Aman Resorts meets a royal Indian palace.

1. Tech stack

- Single self-contained project: vanilla HTML + CSS + JS (no framework, no build step) OR Vite + vanilla if you prefer — but final output must run by opening index.html or a simple static server.
- GSAP + ScrollTrigger (CDN) for all scroll animations, pinning, and scrubbing.
- Lenis (CDN) for buttery smooth-scroll with inertia — this is non-negotiable for the premium feel. Wire Lenis to ScrollTrigger properly (lenis.on('scroll', ScrollTrigger.update)).
- Fonts from Google Fonts: Fraunces (display serif, use optical sizing + light weights for huge headlines) and Inter or Manrope (UI sans). Load with font-display: swap and preload.

2. Design system

- Palette: near-black sapphire background #0A0E1A, ivory text #F5F0E8, royal gold accent #C9A96E, deep teal secondary #1B3A4B. Gold is used sparingly — hairlines, numerals, hover states — never large fills.
- Typography scale: hero headline 8–12vw Fraunces Light with tight tracking; section titles 4–5vw; body max 65ch, 1.7 line-height. Use real typographic details: hanging small-caps eyebrow labels (letter-spacing: 0.35em), old-style numerals for section numbers (01 / 06), thin gold rules.
- Texture: subtle film-grain overlay (CSS or tiny looping noise) at ~3% opacity over everything; soft vignette on video sections.
- Spacing: enormous whitespace. Sections breathe. Nothing cramped.
- Custom cursor: small gold dot that expands into a thin gold ring with the word "SCROLL" or "VIEW" on interactive elements. Disable on touch devices.

3. Scroll architecture (the core of the site)

Use ScrollTrigger pinning so each chapter locks to the viewport while its animation plays out, then releases.

Scroll-scrubbed video technique (clips 01, 02, 06)
- Pin the section for ~300–400vh of scroll distance.
- Map scroll progress → video.currentTime (duration × progress). Use requestAnimationFrame lerp toward the target time (lerp factor ~0.1) so scrubbing feels weighted and cinematic, never jittery.
- Videos must be muted, playsinline, preload="auto". Important: instruct me in the README to re-encode the scrub videos with dense keyframes for smooth seeking, and include the exact ffmpeg command: ffmpeg -i in.mp4 -vf scale=1920:-2 -g 1 -crf 20 -movflags +faststart -an out.mp4.
- Show a minimal gold progress hairline on the side of pinned sections indicating chapter progress.

Autoplay sections (clips 03, 04, 05)
- Lazy-load with IntersectionObserver; play only when ≥50% visible, pause when leaving (battery + performance).
- Video sits behind a dark gradient scrim so overlaid text always passes contrast.
